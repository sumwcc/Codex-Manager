# 最小排障手册

这个手册用于快速定位最常见的启动、转发和模型刷新问题。

## 先看什么

1. 先看 [运行与部署指南](运行与部署指南.md)。
2. 再看 [FAQ 与账号命中规则](FAQ与账号命中规则.md)。

## 常见排查顺序

### 1. 服务是否启动

- 检查桌面端或 service 是否已启动。
- 确认监听地址、端口和代理配置是否正确。
- 看日志里是否有数据库连接失败、端口占用或权限问题。

### 2. 请求是否进入网关

- 确认客户端请求实际打到当前网关地址。
- 看请求日志里是否有对应路径和状态码。
- 如果是第三方客户端，确认 API Key 和 base URL 是否配置正确。

### 3. 模型或账号是否可用

- 检查账号状态是否被禁用、失效或命中限制。
- 刷新用量后再试一次。
- 如果是封禁账号，先走账号清理入口。

### 4. 流式或 tools 异常

- 检查 `tool_calls`、SSE 结束事件、字段还原是否丢失。

## 还不行怎么办

- 直接对照 README 里的文档入口。
- 优先保留出错请求、响应和请求日志，再继续排查。
