# Таблица фильтров учетных записей фоновых задач

Ниже мы рассмотрим только вопрос «где фильтровать какой статус», чтобы облегчить локальное устранение неполадок.

| Задачи | Фильтровать местоположения | Условия фильтра | Пропущенный статус/результаты | Описание |
| --- | --- | --- | --- | --- |
| Ветка опроса использования | [crates/service/src/usage/refresh/batch.rs](../../../crates/service/src/usage/refresh/batch.rs#L91) | `status = disabled` и последнее событие, ставшее причиной бана | `disabled`, `account_deactivated`, `workspace_deactivated` | Сначала поместите эти учетные записи в `skipped_ids`, а затем отфильтруйте их из списка токенов |
| Поток поддержки активности шлюза | [crates/core/src/storage/accounts.rs](../../../crates/core/src/storage/accounts.rs#L117) | `status = active` и соблюсти условия доступности шлюза | Учетные записи, отличные от `active`, не будут напрямую входить в набор кандидатов | Вместо проверки всех учетных записей сначала будут проверены учетные записи кандидатов, а затем сохранены |
| Опрос обновления токена | [crates/core/src/storage/tokens.rs](../../../crates/core/src/storage/tokens.rs#L28) | `refresh_token` не может быть пустым; последний статус не может быть причиной деактивации класса | Нет `refresh_token`, `account_deactivated`, `workspace_deactivated` | Обрабатывать только те токены, которые можно обновить и которые не деактивированы |

## Дополнение

| Феномен | Общие причины |
| --- | --- |
| Кажется, «доступно одну секунду, срок действия истек в следующую секунду» | Сервер не вернется до следующего обновления `refresh_token_expired` |
| Для поддержания активности шлюза не выполняется никаких действий | Учетная запись кандидата пуста или нет доступной учетной записи `active` |
| Никаких изменений в опросах об использовании | Счета были отфильтрованы, или в текущем пакете нет счетов, которые можно было бы обработать |

## Самый краткий вывод

И `用量轮询`, и `令牌刷新` будут фильтровать учетные записи по статусу; `网关保活` распознаёт только аккаунты-кандидаты из `active`.
